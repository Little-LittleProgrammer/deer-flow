# 需求大致规划

## 基础工具

1. 基于 deerflow 此项目拓展
2. 接入飞书项目，飞书项目文档可以通过此 mcp 找到

```json
{
    "mcpServers": {
        "lark-project-dev-docs-mcp": {
            "type": "http",
            "url": "https://project.feishu.cn/mcp_server/knowledge?userKey=7264498293391065091"
        }
    }
}
```

3. 接入飞书文档和消息，npm install -g @larksuite/cli，使用此 cli 接入
4. 接入阿里云 Codeup，作为代码托管平台，支持代码拉取、提交、PR 创建与代码评审联动。

## 整理流程

### 产品规划阶段

1. 选择当前需求，以及选择相关代码仓库，agent 基于仓库和项目需求的飞书文档链接，启用 ultra 模式，分析需求的可行性，所存在问题，整理出评论，通过飞书云文档的评论 api 发布全文评论
2. 需求文档

### 评审阶段

1. 针对当前需求以及相关代码库，启用 ultra 模式，整理 spec 文档、design 文档、task 文档、测试文档，通过飞书 api 创建相关云文档

### 开发阶段

1. 针对当前需求以及相关代码库，整理 spec 文档、design 文档、task 文档、测试文档，启用 ultra 模式，根据相关文档，进行开发

### 测试阶段

1. 针对当前需求以及相关代码库，整理 spec 文档、design 文档、task 文档、测试文档，启用 ultra 模式，根据相关文档，进行测试
